# 跌倒检测系统技术文档

## 1. 项目概述

本项目是一个基于 YOLO 姿态估计的实时跌倒检测系统，通过摄像头采集视频流，利用 YOLO 模型进行人体姿态估计，结合多规则判定算法检测跌倒事件，并通过 WebSocket 和 BLE 两种方式向手机 APP "落影有应" 发送告警。

系统提供两个平台版本：
- **Windows 本地版** (`fall_detect_windows.py`) - 适用于 PC 端开发和测试
- **Raspberry Pi 版** (`fall_detect_raspi.py`) - 适用于边缘部署

---

## 2. 系统架构

```
摄像头帧 → YOLO Pose 推理 → 17 关键点 → FallDetector (5 规则 + 时序验证)
                                                |
                                    [跌倒确认] → JPEG 编码
                                                |
                          ┌─────────────────────┼─────────────────────┐
                          |                                           |
                   WebSocket 广播                              BLE 图片传输
                   (JSON + base64)                            (分包 JPEG)
                          |                                           |
                          └──────────> 手机 APP "落影有应" <──────────┘
```

### 2.1 核心组件

| 组件 | 功能 |
|---|---|
| `FallDetector` | 跌倒检测核心算法，5 条规则 + 时序一致性验证 |
| `WsServer` | WebSocket 服务器，推送告警到手机 APP |
| `BleImageSender` | BLE 图片传输（Windows 客户端模式） |
| `BleGattServer` | BLE GATT 服务端（Raspberry Pi 服务端模式） |
| `FallDetectionSystem` | 主系统类，协调各组件运行 |

### 4.2 检测算法参数配置

`FallDetector` 类的参数可通过代码修改或命令行扩展配置：

#### 规则阈值参数

| 参数 | 默认值 | 说明 |
|---|---|---|
| `aspect_ratio_threshold` | 1.2 | 身体宽高比阈值，超过此值视为水平姿态 |
| `angle_threshold` | 55.0 | 躯干倾斜角度阈值（度） |
| `drop_ratio_threshold` | 0.75 | 头部下降比阈值 |
| `confidence_threshold` | 0.5 | 关键点最低置信度 |
| `min_visible_points` | 6 | 最少可见关键点数 |
| `area_change_threshold` | 2.5 | 边界框面积突变倍数阈值 |

#### 时序参数

| 参数 | 默认值 | 说明 |
|---|---|---|
| `temporal_window` | 8 | 滑动窗口帧数 |
| `min_triggers` | 4 | 窗口内最少触发帧数 |
| `cooldown_frames` | 45 | 跌倒后冷却帧数（约 1.5 秒 @30fps） |

**判定逻辑：**
- 单帧疑似：5 条规则中至少 3 条同时触发
- 时序一致性：滑动窗口内至少 `min_triggers` 帧疑似 → 确认跌倒
- 冷却机制：跌倒确认后 `cooldown_frames` 帧内不再触发

### 4.4 WebSocket 通信机制

#### 4.4.1 WebSocket 的作用

WebSocket 是系统向手机 APP 推送跌倒告警（含现场图片）的**主要传输通道**。与 BLE 蓝牙相比，WebSocket 在局域网内传输速度更快、延迟更低，适合实时告警场景。

核心职责：
- 维护与手机 APP 的**持久双向连接**
- 跌倒确认时**主动推送**告警 JSON（含 base64 编码的 JPEG 图片）
- 支持**多个客户端**同时连接（广播模式）

#### 4.4.2 连接流程

```
手机 APP (落影有应)                         Python 程序 (WsServer)
       |                                          |
       |  1. 手机发起 WebSocket 连接               |
       |  ws://<电脑局域网IP>:8765                  |
       | ─────────────────────────────────────────> |
       |                                          |
       |  2. 连接建立，手机加入 clients 集合         |
       | <───────────────────────────────────────── |
       |                                          |
       |         ... 等待跌倒事件 ...               |
       |                                          |
       |  3. YOLO 检测到跌倒 → JPEG 编码 → Base64   |
       |                                          |
       |  4. 服务端 broadcast() 推送 JSON           |
       | <───────────────────────────────────────── |
       |                                          |
       |  5. 手机 APP 解码 Base64 → 显示图片+告警    |
       |                                          |
```

**手机连接步骤：**
1. 确保手机和运行 Python 程序的设备处于**同一局域网**
2. 手机 APP 自动扫描局域网内的 WebSocket 服务，或手动输入 `ws://<设备IP>:8765`
3. 连接建立后进入监听状态，无需发送任何数据（单向接收）

#### 4.4.3 服务端实现

WebSocket 服务器基于 `websockets` 库实现，运行在**独立线程**中，不阻塞主检测循环：

```python
# fall_detect_windows.py:275-308
class WsServer:
    def __init__(self, host="0.0.0.0", port=8765):
        self.clients: set = set()          # 已连接的客户端集合

    async def _handler(self, websocket, path=None):
        self.clients.add(websocket)        # 客户端连接时加入集合
        try:
            async for _ in websocket:      # 保持连接，无需客户端发送数据
                pass
        finally:
            self.clients.discard(websocket) # 断开时移除

    def broadcast(self, message: dict):
        """从任意线程发送消息到所有客户端"""
        data = json.dumps(message)
        for ws in list(self.clients):
            asyncio.run_coroutine_threadsafe(ws.send(data), self._loop)
```

启动方式（在 `FallDetectionSystem.run()` 中）：
```python
ws_loop = asyncio.new_event_loop()
ws_thread = threading.Thread(target=self._run_ws_server, args=(ws_loop,), daemon=True)
ws_thread.start()
```

#### 4.4.4 告警消息格式

跌倒确认时，系统将当前帧编码为 JPEG（质量 70%），Base64 编码后封装为 JSON 广播：

```json
{
    "type": "fall_alert",
    "timestamp": "2026-05-06 14:30:00",
    "image": "/9j/4AAQSkZJRgABAQAAAQABAAD/2wBD...",
    "rules": {
        "aspect_ratio": {"triggered": true, "value": 1.5, "threshold": 1.2},
        "trunk_angle":  {"triggered": true, "value": 65.0, "threshold": 55.0},
        "head_drop":    {"triggered": true, "value": 0.6, "threshold": 0.75}
    },
    "confidence": 0.85,
    "visible_points": 12
}
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `type` | string | 消息类型，固定为 `"fall_alert"` |
| `timestamp` | string | 告警时间，格式 `YYYY-MM-DD HH:MM:SS` |
| `image` | string | Base64 编码的 JPEG 图片（质量 70%） |
| `rules` | object | 本次触发的规则详情（仅包含 `triggered: true` 的规则） |
| `confidence` | float | 关键点平均置信度 (0~1) |
| `visible_points` | int | 可见关键点数量 (0~17) |

#### 4.4.5 配置参数

```bash
python fall_detect_windows.py --ws-host 0.0.0.0 --ws-port 8765
```

| 参数 | 默认值 | 说明 |
|---|---|---|
| `--ws-host` | `0.0.0.0` | 监听地址，`0.0.0.0` 表示接受所有网卡连接 |
| `--ws-port` | `8765` | 监听端口 |

**注意事项：**
- 若设备开启了防火墙，需放行对应端口
- 多网卡环境下，手机需连接到与 Python 程序同一网段的网络
- 告警推送有 3 秒冷却时间，避免短时间内重复发送

### 4.5 BLE 配置

#### Windows 版（客户端模式）

BLE 客户端会扫描并连接名称包含指定前缀的设备：

```bash
python fall_detect_windows.py --ble-device "FallDetect"
```

#### Raspberry Pi 版（服务端模式）

Raspberry Pi 作为 BLE GATT 服务器，手机 APP 主动连接。

**BLE 协议常量：**
| 常量 | UUID | 说明 |
|---|---|---|
| SERVICE_UUID | 12345678-1234-5678-1234-56789abcdef0 | 主服务 |
| CHAR_IMAGE_UUID | 12345678-1234-5678-1234-56789abcdef1 | 图片数据特征 |
| CHAR_CONTROL_UUID | 12345678-1234-5678-1234-56789abcdef2 | 控制指令特征 |
| CHAR_STATUS_UUID | 12345678-1234-5678-1234-56789abcdef3 | 状态特征 |

**数据包格式：**
- 包头：4 字节 (`flags` + `reserved` + `sequence`)
- 数据：最大 496 字节 JPEG 数据
- flags bit0 = 首包，bit1 = 末包

## 5. 使用说明

### 5.1 快速启动


## 6. 高级配置

### 6.1 调整检测灵敏度

修改 `FallDetector` 参数可调整检测灵敏度：

```python
# 在代码中修改
detector = FallDetector(
    aspect_ratio_threshold=1.0,   # 降低阈值，更容易触发
    angle_threshold=45.0,         # 降低角度阈值
    min_triggers=3,               # 降低时序触发要求

```

**灵敏度调整建议：**
- 降低阈值 → 更灵敏，但可能增加误报
- 提高阈值 → 更严格，但可能漏检

## 7. 故障排除

### 7.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|---|---|---|
| 无法打开摄像头 | 摄像头被占用或不存在 | 检查摄像头 ID，关闭其他程序 |
| 模型加载失败 | 模型文件不存在 | 确认模型文件在当前目录 |
| WebSocket 连接失败 | 端口被占用或防火墙 | 更换端口或检查防火墙设置 |
| BLE 连接失败 | 设备未开启或距离过远 | 检查 BLE 设备状态 |
| FPS 过低 | 硬件性能不足 | 降低分辨率或使用更轻量模型 |

### 7.2 日志查看

系统使用 Python logging 模块输出日志，可通过修改日志级别查看详细信息：

```python
logging.basicConfig(level=logging.DEBUG)  # 改为 DEBUG 级别
```
